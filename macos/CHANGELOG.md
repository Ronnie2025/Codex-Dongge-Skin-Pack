# Changelog

## 1.3.0 — 2026-07-16

### 新增

- 同包内置「语言的用法」「开源工作台」「心理问题举牌」三套可切换栋哥主题，并保留经典白板兼容主题
- 安装后生成 `~/Applications/栋哥 Codex.app` 与桌面入口，日常启动不再依赖 `.command` 文件
- 移除自动修复与宿主自动重启：如果官方 Codex 已打开但缺少皮肤 CDP 端口，脚本会提示手动退出
- 引擎从 `~/.codex` 迁到 `~/Library/Application Support/CodexDreamSkinStudio/engine`，避免新版 Codex 将运行脚本隔离为 `scripts.DISABLED`
- 新增统一 Mac / Windows 打包脚本 `scripts/build-unified-release.sh`

### 改进

- Pause / Restore 不再重启宿主，`--uninstall` 会删除桌面入口与栋哥 Codex.app
- Windows 与 macOS 共用主题配置结构，支持三套主题切换和同一套动态配色文案
- 继续只使用 `127.0.0.1` CDP，不修改官方 `.app`、`app.asar`、WindowsApps 或模型供应商设置

## 1.1.0 — 2026-07-16

- SwiftBar 菜单栏入口（`Install Menu Bar.command`）：应用 / 暂停 / 换图 / 切换已保存主题 / 从图片文件夹加载 / 完全恢复

- 安装时不再强制 `appearanceTheme=dark`，只备份桌面外观相关键，便于恢复与自动适配

- 以原版暗色 portal CSS 为结构底，叠加 light 壳与更薄横幅遮罩，减轻「换图看不清」

## 1.0.0 — 2026-07-15

- 发布 macOS 通用主题制作器，而不是固定角色皮肤。
- 加入 Finder 选图、自动 JPEG 转换、主题命名和高级配色参数。

- 增加独立安装目录、桌面启动/定制/验证/恢复入口。
- 增加官方签名、CDP 端口归属、PID 身份、刷新重注入和真实 DOM 自检。
- 增加原子配置备份、精确恢复、静态测试、安装恢复循环和发布打包脚本。
- 清理固定角色内部命名；传送门主题仅作为可替换示例素材。
- 开源树：示例横幅改为无角色抽象几何图；验收截图不入库；补充 NOTICE / README 商标与安全边界说明。
